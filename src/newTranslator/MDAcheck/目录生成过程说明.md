# MDAcheck 目录生成过程说明

本文档说明 `/root/code/ChainCollab/src/newTranslator/MDAcheck` 中几个主要文件夹和产物是如何一步步生成出来的，重点按“执行什么命令、做了什么、产物落在哪个目录”来描述。

## 1. 导出 Ecore 元模型

先在项目根目录执行：

```bash
cd /root/code/ChainCollab/src/newTranslator
./.venv/bin/python MDAcheck/export_b2c_ecore.py
```

这一步会读取 B2CDSL 的 textX 语法文件 `b2c.tx`，导出 EMF 可识别的元模型 `b2c.ecore`。

产物位置：

- `MDAcheck/b2c.ecore`

作用：

- 后续随机实例生成依赖这份元模型
- `.b2c -> .xmi` 转换依赖这份元模型
- Eclipse OCL 校验也依赖这份元模型

## 2. 将单个 `.b2c` 实例转换为 `.xmi`

如果输入是 DSL 文本实例，可以执行：

```bash
cd /root/code/ChainCollab/src/newTranslator
./.venv/bin/python MDAcheck/b2c_to_xmi.py \
  --in build/b2c/chaincode.b2c \
  --out MDAcheck/chaincode.xmi
```

这一步会：

- 用 textX 解析 `.b2c`
- 用 `b2c.ecore` 创建对应的 EMF 对象
- 输出成 Eclipse/EMF 可校验的 `.xmi`

产物位置：

- 默认是 `MDAcheck/chaincode.xmi`
- 如果改了 `--out`，则输出到你指定的位置

## 3. 生成大量随机 XMI 模型

随机模型生成使用的是 `MDAcheck/emf-random/` 下的 instantiator。

推荐执行：

```bash
cd /root/code/ChainCollab/src/newTranslator/MDAcheck/emf-random
./run_instantiator.sh -m ../b2c.ecore -n 100 -s 1000 -e 12345
```

参数含义可按当前实验理解为：

- `-m ../b2c.ecore`：指定元模型
- `-n 100`：生成 100 个随机模型
- `-s 1000`：生成规模或搜索步数
- `-e 12345`：随机种子

这一步生成的是大量随机的 `.xmi` 实例模型。

在当前目录结构中，这批随机模型产物位于：

- `MDAcheck/emf-random/b2c-100/b2c/`

目录内部一般是这种形式：

```text
emf-random/b2c-100/b2c/model0/result0_200.xmi
emf-random/b2c-100/b2c/model1/result1_200.xmi
...
emf-random/b2c-100/b2c/model99/result99_200.xmi
```

也就是说：

- `emf-random/` 是随机实例生成工具目录
- `emf-random/b2c-100/b2c/` 是一批随机生成结果的输出目录

## 4. 对随机生成结果批量做 OCL 校验

对整个随机结果目录批量执行 OCL 校验，可运行：

```bash
cd /root/code/ChainCollab/src/newTranslator
bash MDAcheck/run_ocl_validate_batch.sh \
  MDAcheck/emf-random/b2c-100/b2c \
  MDAcheck/b2c.ecore \
  MDAcheck/check.ocl \
  MDAcheck/ocl-report-random-100
```

这一步会：

- 遍历输入目录下所有 `.xmi`
- 调用 `ocl-runner-tycho` 中的 Eclipse OCL 校验器
- 汇总每个模型的违反规则情况
- 生成总报告和逐模型报告

产物位置：

- `MDAcheck/ocl-report-random-100/report.json`
- `MDAcheck/ocl-report-random-100/report.md`
- `MDAcheck/ocl-report-random-100/violation_profile.md`
- `MDAcheck/ocl-report-random-100/models/`

其中：

- `report.md` 是总览表
- `violation_profile.md` 是规则级统计
- `models/` 里是每个随机模型的单独 JSON 报告

## 5. 正例数据集目录是怎么来的

`bpmn-positive/` 表示一组人工准备或已有的正例模型。

目录结构：

- `MDAcheck/bpmn-positive/b2c/`
- `MDAcheck/bpmn-positive/xmi/`

含义：

- `b2c/` 存放正例 DSL 文本
- `xmi/` 存放这些正例转换后的 XMI

生成方式一般是：

1. 先准备 `.b2c` 文件到 `bpmn-positive/b2c/`
2. 再用 `b2c_to_xmi.py` 批量或逐个转成 `.xmi`
3. 输出到 `bpmn-positive/xmi/`

这些正例经过批量校验后，报告输出到：

- `MDAcheck/ocl-report-bpmn-positives/`

## 6. 变异数据集目录是怎么来的

`bpmn-mutations/` 是在正例 XMI 基础上，人工注入特定错误后生成的一批“故意违规”的模型。

生成命令对应脚本：

```bash
cd /root/code/ChainCollab/src/newTranslator/MDAcheck
python bpmn-mutations/generate_mutation_failures.py
```

这一步会：

1. 读取 `bpmn-positive/xmi/` 中的正例模型
2. 对每个模型应用几种固定变异
3. 把变异后的 XMI 写到 `bpmn-mutations/xmi/`
4. 自动调用批量 OCL 校验
5. 生成变异报告

当前脚本里的典型变异包括：

- 让 message 的 `sender == receiver`
- 让 participant 的 `multiMin > multiMax`
- 让两个 participant 重名

产物位置：

- `MDAcheck/bpmn-mutations/xmi/`
- `MDAcheck/bpmn-mutations/manifest.pre-validation.json`
- `MDAcheck/bpmn-mutations/manifest.json`
- `MDAcheck/ocl-report-bpmn-mutations/`

其中：

- `manifest.pre-validation.json` 记录变异计划
- `manifest.json` 记录变异后实际校验结果
- `ocl-report-bpmn-mutations/` 是这批变异模型的校验报告目录

## 7. 单个模型的一键校验流程

如果只是想对一个 `.b2c` 做完整链路验证，可以直接执行：

```bash
cd /root/code/ChainCollab/src/newTranslator
bash MDAcheck/run_ocl_validate.sh
```

它内部顺序是：

1. 导出 `b2c.ecore`
2. 把 `build/b2c/chaincode.b2c` 转成 `MDAcheck/chaincode.xmi`
3. 用 `check.ocl` 对 `chaincode.xmi` 做校验

主要产物：

- `MDAcheck/b2c.ecore`
- `MDAcheck/chaincode.xmi`

这条命令更适合单模型调试，不是批量实验入口。

## 8. `ocl-runner-tycho/` 的作用

`MDAcheck/ocl-runner-tycho/` 不是数据产物目录，而是 OCL 校验器本身的工程目录。

它的作用是：

- 通过 Tycho + Eclipse OCL Bundle 运行 Complete OCL 校验
- 被 `run_ocl_validate.sh` 和 `run_ocl_validate_batch.sh` 调用

它本身主要是执行工具，不是实验结果输出目录。

## 9. 目录用途总览

可以把整个流程概括为：

```text
b2c.tx
  -> export_b2c_ecore.py
  -> MDAcheck/b2c.ecore

.b2c 正例
  -> b2c_to_xmi.py
  -> MDAcheck/bpmn-positive/xmi/

b2c.ecore
  -> emf-random/run_instantiator.sh
  -> MDAcheck/emf-random/b2c-100/b2c/

随机/正例/变异 XMI
  -> run_ocl_validate_batch.sh
  -> MDAcheck/ocl-report-*/
```

对应关系可以记成：

- `emf-random/`：随机生成器
- `bpmn-positive/`：人工正例集
- `bpmn-mutations/`：从正例注入错误得到的违规集
- `ocl-report-*`：对应数据集的校验结果
- `ocl-runner-tycho/`：真正执行 Eclipse OCL 的工具工程

## 11. 一句话总结

`MDAcheck` 这套目录本质上是在做一条完整链路：

`textX 语法 -> Ecore 元模型 -> XMI 实例生成/转换 -> Eclipse OCL 批量校验 -> 报告输出`

如果你只关注“随机生成的大量模型在哪”，答案就是：

- `MDAcheck/emf-random/b2c-100/b2c/`

如果你只关注“随机模型校验报告在哪”，答案就是：

- `MDAcheck/ocl-report-random-100/`
