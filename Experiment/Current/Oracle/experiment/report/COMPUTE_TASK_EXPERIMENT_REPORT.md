# Oracle 计算任务实验报告

生成时间：2026-03-04T13:20:25.717Z

## 1. 实验目标

- 验证四类计算任务（简单/浮点/随机/复杂）在链上与链下两条路径下的开销差异。
- 验证“链下有额外开销，但适用于浮点、随机及复杂运算”这一结论。

## 2. 计算任务类型

- 类型总数：4；任务总数：4。
- 类型包括：简单运算、浮点运算、随机运算、复杂运算。
- 实验输入文件：`dataset/compute_tasks_10_scenarios.json`。

## 3. 计算任务开销结果

| 任务类型 | 代表计算任务 | 复杂性说明 | 链上计算Cost(gas) | 链下计算Cost(gas) | 优化率 |
| --- | --- | --- | ---: | ---: | ---: |
| 简单运算 | 线性结算公式：amount = a * b + c | 常数复杂度，约 1 次乘法 + 1 次加法 | 46,183 | 72,033 | -55.97% |
| 浮点运算 | 连续非线性评分：score = α·log(x+1) + β·exp(y) + γ·sqrt(z) | 含 log/exp/sqrt 等连续函数，需浮点支持 | N/A（链上不支持） | 54,933 | N/A |
| 随机运算 | 蒙特卡洛概率估计：p ≈ hits / N | N 轮抽样循环（本实验 N=4000）+ 随机数比较与计数 | N/A（链上不支持） | 54,933 | N/A |
| 复杂运算 | 方案组合优化：result=min[(60x+25y+15z-target)^2 + 2*(400x+120y+80z-budget)^2] | 离散搜索优化（本实验 11×11×11=1331 组），每组计算分值偏差+预算偏差惩罚 | 2,263,134 | 54,933 | 97.57% |

## 4. 结果分析

1. 在可直接链上计算的 2 个场景中，链下路径在 1 个场景中更省开销。
2. 简单运算场景中，链下提交的固定成本明显，整体呈现额外开销。
3. 浮点运算与随机运算场景在当前 EVM 口径下不适合直接链上实现，链下执行是必要路径。
4. 复杂运算场景中，链下计算 + 链上确认可显著降低开销，体现出明显收益。
5. 因此可得结论：链下计算存在相当的额外开销，但适用于浮点、随机及复杂运算场景。
