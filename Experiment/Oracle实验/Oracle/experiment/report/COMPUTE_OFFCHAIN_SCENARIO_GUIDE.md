# 计算任务链下执行场景构思（可直接用于论文）

## 一、必须链下执行（链上不可等价表达）

这些任务的核心特征是：依赖浮点、指数、对数、三角函数或高阶统计函数，在 EVM 整数算术模型下难以无损表达。

1. 连续健康指数评估（设备健康评分）
- 场景：水处理/化工设备状态评估
- 典型表达：
```js
score = 0.42*Math.log(vibration+1) + 0.35*Math.exp(temperature/100) + 0.23*Math.sqrt(load)
```
- 链下必要性：`log/exp/sqrt` 不适合直接链上原生实现。

2. 连续风险函数（工艺风险概率）
- 场景：供水网络/流程工业风险告警
- 典型表达：
```js
z = 0.015*pressure + 0.02*flow - 0.01*ph + 0.5*Math.sin(vibration/10)
risk = 1/(1+Math.exp(-z))
```
- 链下必要性：`sin` 和 logistic 函数不适合直接链上表达。

3. 滤波类任务（例如卡尔曼/粒子滤波）
- 场景：多传感器融合与实时状态估计
- 链下必要性：矩阵运算 + 浮点稳定性要求高，链上实现代价大且精度受限。

## 二、非常复杂且更适合链下执行（链上可做但成本高）

这些任务在理论上可用整数近似实现，但链上 gas 成本会快速上升。

1. 多约束滚动排产优化
- 场景：钢铁、半导体产线班次排产
- 典型表达：多轮迭代 + 目标函数评估 + 约束惩罚。
- 适合链下原因：迭代次数增加时，链上成本近线性/超线性增长。

2. 维修计划优化（预测性维护）
- 场景：发动机、泵组、压缩机维护窗口决策
- 典型表达：以停机损失、备件成本、风险成本构成目标函数的多轮搜索。
- 适合链下原因：多参数联合搜索导致链上开销高。

3. 多阶段质量门控与策略搜索
- 场景：离散制造多工序质量控制
- 典型表达：多阶段决策树/规则链 + 组合路径搜索。
- 适合链下原因：规则组合数增大后链上执行成本显著上升。

## 三、仍建议链上执行（简单且确定）

这些任务固定成本低、可解释性强，链下提交反而可能更贵。

1. 结算类线性公式（energy * tariff + penalty）
2. 比率类 KPI（FPY/Availability/Quality/OEE 的整数化表达）
3. 固定阈值逻辑判断（pass/fail、报警开关）

## 四、论文可用结论模板

1. 简单确定性任务（线性/比率/阈值）优先链上执行。  
2. 连续非线性任务（浮点/对数/指数/三角）属于链下必要执行。  
3. 高复杂优化任务链上虽可实现，但成本高，宜采用链下计算 + 链上确认。  
4. 因此应采用“按可表达性 + 复杂度”的混合选路策略。
